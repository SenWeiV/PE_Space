# 内部工具托管平台 - 功能文档

> 版本：v1.0 | 更新时间：2026-03-02

## 产品概述

面向内部团队的 Streamlit App 自动托管平台，类似私有化部署的 HuggingFace Spaces。PM/PE/RD 将写好的 Python Streamlit 工具打包为 zip 上传后，平台自动完成容器化部署，生成可分享的内网 URL，无需具备服务器运维能力。

---

## 一、用户与权限

### 1.1 账号体系

| 角色 | 说明 |
|------|------|
| `admin` | 管理员，拥有所有权限 |
| `user` | 普通用户，可上传/管理自己的 App |

- 账号需由管理员创建，不支持自助注册
- 支持账号启用/停用（停用后无法登录）

### 1.2 登录认证

- 用户名 + 密码登录，密码使用 bcrypt 哈希存储
- 登录成功后颁发 JWT Token（默认有效期 24 小时）
- Token 存储于 localStorage，页面刷新后自动保持登录状态
- Token 失效或未登录时，自动跳转至登录页

## 二、App 管理

### 2.1 上传 & 部署（核心功能）

**三步上传向导：**

**第一步：填写基本信息**
- App 名称（展示用）
- Slug（URL 路径，如 `data-analyzer`，3-64 位小写字母/数字/连字符）
- 描述（可选）

**第二步：上传 zip 包**
- 支持拖拽上传
- zip 包要求：
  - 必须包含 `app.py`（Streamlit 入口文件）
  - 必须包含 `requirements.txt`（依赖列表）
  - 平台自动识别 macOS 打包产生的嵌套目录结构

**第三步：确认部署**
- 确认后后台自动执行：
  1. 安全解压 zip，防止路径穿越攻击
  2. 平台注入标准 Dockerfile（用户无需自己写）
  3. 执行 `docker build`，完整捕获构建日志
  4. 动态分配宿主机端口（8600-9600 范围，自动避免冲突）
  5. 启动容器（`unless-stopped` 重启策略）
  6. 写入 Traefik 路由配置，立即可访问

**注入的标准 Dockerfile（用户无感知）：**
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8501
CMD ["streamlit", "run", "app.py", "--server.port=8501",
     "--server.address=0.0.0.0", "--server.headless=true",
     "--server.baseUrlPath=/apps/{slug}"]
```

### 2.2 App 状态流转

```
pending（待上传）
    ↓ 上传 zip
building（构建中）
    ↓ 构建成功           ↓ 构建失败
running（运行中）      failed（构建失败）
    ↓ 停止
stopped（已停止）
    ↓ 启动
running（运行中）
```

### 2.3 App 操作

| 操作 | 触发条件 | 说明 |
|------|----------|------|
| 访问 App | 状态为 running | 在新标签页打开 `/apps/{slug}` |
| 停止 | 状态为 running | 停止容器 |
| 启动 | 状态为 stopped | 重启已停止的容器 |
| 删除 | 自己的 App 或管理员 | 停止并删除容器、路由配置、上传文件、数据库记录 |

### 2.4 构建日志

- App 详情页可查看完整构建日志
- 构建中时每 3 秒自动刷新日志
- 构建失败时保留完整错误信息（含 pip install 报错）
- 日志以等宽字体深色背景展示

### 2.5 App 列表

- 卡片式网格布局，响应式自适应
- 鼠标悬停显示操作按钮
- 状态指示灯（绿色脉冲 = 运行中）
- 每 10 秒自动轮询刷新状态

## 五、基础设施

### 5.1 容器隔离

- 每个 App 运行在独立 Docker 容器中，互不干扰
- 容器命名规则：`app_{id}_{slug}`
- 镜像命名规则：`tool-platform/app-{id}:{slug}`
- 容器异常退出后自动重启（`unless-stopped`）

### 5.2 反向代理路由

| 路径 | 目标 | 优先级 |
|------|------|--------|
| `/api/*` | FastAPI 后端 | 10（最高） |
| `/apps/{slug}/*` | 对应 App 容器 | 5 |
| `/*` | React 前端 | 1（兜底） |

- 使用 Traefik 动态路由，App 部署后无需重启代理即可生效
- 每个 App 路由配置存储于独立 YAML 文件，删除 App 时同步清理

### 5.3 端口管理

- App 容器端口从 8600-9600 动态分配
- 分配前自动检查 Docker 层面 + socket 层面的端口占用
- 保证多 App 并发部署不产生端口冲突

## 七、App 开发规范（给 Vibe Coding 开发者）

### 7.1 数据存储机制（必读）

每个 App 运行在独立 Docker 容器中，工作目录是 `/app/`。理解数据的生命周期非常重要：

| 数据写入位置 | 容器重启 | 重新部署 | 删除 App |
|---|---|---|---|
| 容器内 `/app/` 任意文件 | ✅ 保留 | ❌ **丢失** | ❌ 丢失 |
| `st.session_state` | ❌ 丢失 | ❌ 丢失 | ❌ 丢失 |

> ⚠️ **重新部署 = 旧容器销毁 + 新容器创建**，容器内所有运行时生成的文件会清空。
> 如果你重新上传了代码，之前跑出来的结果文件就会消失。

**结论：不要依赖容器文件系统来长期保存数据，用下载按钮把结果给到用户。**

---

### 7.2 数据处理的推荐模式

#### ✅ 模式一：处理完立即下载（推荐，适合 90% 场景）

用户上传文件 → App 处理 → 用户下载结果。数据不落盘，处理完即走。

```python
import streamlit as st
import pandas as pd
from io import BytesIO

uploaded = st.file_uploader("上传数据文件", type=["xlsx", "csv"])

if uploaded:
    df = pd.read_excel(uploaded, engine="openpyxl")

    # ... 你的处理逻辑 ...
    result_df = process(df)

    # 生成下载按钮，结果直接在内存里，不写磁盘
    buf = BytesIO()
    result_df.to_excel(buf, index=False, engine="openpyxl")
    st.download_button(
        label="📥 下载结果",
        data=buf.getvalue(),
        file_name="result.xlsx",
        mime="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
    )
```

#### ✅ 模式二：需要在同一会话内多步操作（用 session_state）

同一个用户在当前浏览器会话内，多个步骤共享中间结果，不需要持久化。

```python
import streamlit as st

if "result" not in st.session_state:
    st.session_state.result = None

if st.button("第一步：处理"):
    st.session_state.result = run_step1()

if st.session_state.result is not None:
    if st.button("第二步：导出"):
        export(st.session_state.result)
```

> ⚠️ `session_state` 在用户刷新页面或服务重启后会清空，不能用于跨会话存储。

#### ⚠️ 模式三：需要保留历史记录（写文件到 `/app/data/`，重部署会丢失）

如果需要在容器运行期间积累历史（比如记录每次调用的日志），可以写到 `/app/data/` 目录。
这些文件在容器重启后仍然存在，但**重新部署后会丢失**。

```python
import streamlit as st
from pathlib import Path
import json
from datetime import datetime

DATA_DIR = Path("/app/data")
DATA_DIR.mkdir(exist_ok=True)  # 确保目录存在

def save_result(result: dict):
    filename = DATA_DIR / f"result_{datetime.now().strftime('%Y%m%d_%H%M%S')}.json"
    filename.write_text(json.dumps(result, ensure_ascii=False, indent=2))

def list_results():
    return sorted(DATA_DIR.glob("*.json"), reverse=True)
```

> 📌 如果你的工具确实需要长期持久化存储，联系管理员为该 App 挂载持久化卷。

```python
# ── 侧边栏放参数配置 ──

    st.header("参数配置")
    param = st.selectbox("选择模式", ["模式A", "模式B"])

# ── 主内容区 ──
# ... 核心逻辑 ...
```

#### 耗时任务要有进度反馈

```python
with st.spinner("正在处理，请稍候..."):
    result = long_running_task()

st.success("✅ 处理完成！")
```

对于批量处理，使用进度条：

```python
progress = st.progress(0)
status = st.empty()

for i, item in enumerate(items):
    process(item)
    progress.progress((i + 1) / len(items))
    status.text(f"正在处理 {i+1}/{len(items)}: {item}")

status.text("完成！")
```

#### 错误处理

```python
try:
    result = process(data)
    st.success("处理成功")
except Exception as e:
    st.error(f"处理失败：{e}")
    st.stop()  # 阻止后续代码执行
```

#### 避免重复计算（用缓存）

```python

    # 加载耗时数据
    ...

@st.cache_resource          # 缓存模型/连接等资源（不随参数变化）
def load_model():
    ...
```

### 7.4 配置与密钥管理

#### 方式一：配置文件打包进 zip（简单，适合内部工具）

将 `config.json` 或 `.env` 放在项目目录里一起打包。

```python
import json
from pathlib import Path

config = json.loads((Path(__file__).parent / "config.json").read_text())
api_key = config["api_key"]
```

> ⚠️ 密钥会被打包进 Docker 镜像。内部工具可接受，对外分享时注意风险。

#### 方式二：使用 Streamlit Secrets（推荐，密钥不进镜像）

在项目目录创建 `.streamlit/secrets.toml`，一起打包进 zip：

```toml
# .streamlit/secrets.toml
api_key = "sk-xxxx"
base_url = "https://api.example.com"
```

代码中读取：

```python
api_key = st.secrets["api_key"]
base_url = st.secrets["base_url"]
```

> ✅ 此方式密钥只在运行时读取，不会硬编码在 Python 文件里，相对更安全。

### 7.5 requirements.txt 规范

**必须包含的包：**
```
streamlit>=1.30.0
```

**常用包推荐版本：**
```
# 数据处理
pandas>=2.0.0
openpyxl>=3.1.0      # Excel 读写，替代 calamine
numpy>=1.24.0

# HTTP 请求
requests>=2.31.0
httpx>=0.24.0

# AI/LLM
openai>=1.3.0

# 工具
python-dotenv>=1.0.0
```

**已知不可用的包（Linux arm64 无预编译版本）：**

| 禁用包 | 替代方案 |
|--------|----------|
| `calamine` | `openpyxl`（`pd.read_excel(..., engine="openpyxl")`） |
| `pywin32` | 不适用（Windows 专用） |
| `win32com` | 不适用（Windows 专用） |

**不要写入 requirements.txt 的内容：**
- 本地路径依赖（`-e ./local_package`）
- `git+https://...` 格式（在受限网络环境下可能无法访问）
- 非常规包索引（`--extra-index-url`，除非服务器已配置对应镜像源）

### 7.6 文件结构规范

**最简结构（推荐）：**

```
your-app/
├── app.py              # ✅ 必须，Streamlit 入口
├── requirements.txt    # ✅ 必须
└── config.json         # 可选，配置文件
```

**多模块结构：**

```
your-app/
├── app.py              # ✅ 入口，尽量保持简洁
├── requirements.txt    # ✅ 必须
├── config.json
├── .streamlit/
│   └── secrets.toml    # 密钥（推荐）
└── utils/
    ├── __init__.py
    ├── processor.py    # 核心处理逻辑
    └── helpers.py
```

**不要打包进 zip 的内容：**

```
❌ .DS_Store、__MACOSX/（macOS 垃圾文件）
❌ __pycache__/、*.pyc（Python 缓存）
❌ .venv/、venv/、node_modules/（虚拟环境）
❌ Dockerfile（平台会自动注入）
❌ results/、output/（历史运行结果，容器重建后会丢失，意义不大）
❌ *.log（日志文件）
```

### 7.7 打包命令

```bash
# 在项目目录内执行（不是在上层目录）
cd your-app-folder

# 推荐：排除垃圾文件后打包
zip -r ../your-app.zip . \
  --exclude "*/\.*" \
  --exclude "*/__pycache__/*" \
  --exclude "*/*.pyc" \
  --exclude "*/venv/*" \
  --exclude "*/.venv/*" \
  --exclude "*/results/*" \
  --exclude "*/output/*"
```

> ⚠️ 一定要在**项目目录内**执行 `zip`，不要在上层目录执行 `zip -r app.zip your-app-folder/`。
> 后者会产生一层额外的目录嵌套（平台虽然能处理 3 层，但更容易出问题）。

### 7.8 上传前自测清单

```
□ 本地 python app.py 能正常启动（无语法错误）
□ requirements.txt 列出了所有第三方依赖（不依赖本地已装但未声明的包）
□ 没有使用 Windows 专用包或 arm64 不兼容包
□ 没有硬编码 localhost / 127.0.0.1 端口（这些在容器里不可用）
□ 文件读写路径使用相对路径或 Path(__file__).parent，不要写绝对路径
□ 大文件（模型文件、数据集）不打包进 zip，改用运行时下载或联系管理员挂载
```
