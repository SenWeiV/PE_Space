## 2. 安全组配置（创建实例时配置）

### 2.1 必开端口

在云服务控制台创建安全组，添加以下规则：

| 端口   | 协议 | 源地址        | 用途               | 优先级 |
| ------ | ---- | ------------- | ------------------ | ------ |
| `22`   | TCP  | `[本机IP]/32` | SSH 登录           | 高     |
| `80`   | TCP  | `0.0.0.0/0`   | HTTP 访问          | 中     |
| `443`  | TCP  | `0.0.0.0/0`   | HTTPS 访问         | 中     |
| `____` | TCP  | `0.0.0.0/0`   | 应用端口（如8000） | 中     |

> **注意**：`[本机IP]/32` 表示仅允许你的电脑访问 SSH，更安全

### 2.2 可选端口（根据项目需要）

| 端口        | 协议 | 源地址       | 用途                |
| ----------- | ---- | ------------ | ------------------- |
| `3306`      | TCP  | `10.0.0.0/8` | MySQL（仅内网）     |
| `6379`      | TCP  | `10.0.0.0/8` | Redis（仅内网）     |
| `3000`      | TCP  | `0.0.0.0/0`  | Node 开发端口       |
| `8000-9000` | TCP  | `0.0.0.0/0`  | Python 开发端口范围 |

## 4. 快速部署指南

### 步骤 1：获取本机公网 IP

在本机终端执行：

```bash
curl -s ip.sb
```

将结果填入第1节「本机公网 IP」处，格式：`xxx.xxx.xxx.xxx/32`

### 步骤 2：创建云服务器实例

1. 登录云服务商控制台（阿里云/腾讯云/百度云等）
2. 进入「云服务器」→「创建实例」
3. 按以下配置选择：
   - **镜像**：Ubuntu 22.04 LTS
   - **规格**：2核4G（最低配置）
   - **系统盘**：40GB SSD
   - **公网 IP**：勾选「分配公网 IP」
   - **安全组**：选择或创建安全组（见第2节）
   - **登录方式**：密码 或 SSH Key
4. 创建完成后，将「公网 IP」填入第1节

### 步骤 3：连接服务器并初始化

```bash
# SSH 连接（使用密码）
ssh root@[公网IP]

# 或使用 SSH Key
ssh -i [SSH Key路径] root@[公网IP]

# 在服务器上执行初始化
apt update && apt upgrade -y
apt install -y curl wget vim git htop

# 创建项目目录
mkdir -p /opt/[项目名称]
```

### 步骤 4：打包并上传项目

在本机执行：

```bash
# 进入项目目录
cd [项目本地路径]

# 打包（排除依赖和缓存）
tar \
  --exclude='.git' \
  --exclude='node_modules' \
  --exclude='.venv' \
  --exclude='venv' \
  --exclude='dist' \
  --exclude='build' \
  --exclude='__pycache__' \
  --exclude='*.log' \
  -czf deploy.tar.gz \
  .

# 上传到服务器
scp deploy.tar.gz root@[公网IP]:/opt/[项目名称]/

# 或指定端口
scp -P [SSH端口] deploy.tar.gz root@[公网IP]:/opt/[项目名称]/
```

### 步骤 5：在服务器解压并安装依赖

```bash
# SSH 到服务器
ssh root@[公网IP]

# 进入项目目录
cd /opt/[项目名称]

# 解压
tar -xzf deploy.tar.gz

# 根据项目类型安装依赖
# Python 项目：
python3 -m venv venv
source venv/bin/activate
pip install -r [依赖文件]

# Node 项目：
# npm install
# npm run build

# 配置环境变量
cp .env.example .env
vim .env  # 编辑配置
```

### 步骤 6：配置 systemd 常驻服务

创建服务文件：

```bash
cat > /etc/systemd/system/[项目名称].service << 'EOF'
[Unit]
Description=[项目名称]
After=network-online.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/[项目名称]
ExecStart=[启动命令的完整路径]
Restart=always
RestartSec=3
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
EOF
```

启动服务：

```bash
systemctl daemon-reload
systemctl start [项目名称]
systemctl enable [项目名称]
systemctl status [项目名称]
```

### 步骤 7：配置 Nginx 反向代理（推荐）

```bash
# 安装 Nginx
apt install nginx

# 创建配置
cat > /etc/nginx/sites-available/[项目名称] << 'EOF'
server {
    listen 80;
    server_name [公网IP];

    location / {
        proxy_pass http://127.0.0.1:[服务端口号];
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

# 启用配置
ln -s /etc/nginx/sites-available/[项目名称] /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

### 步骤 8：验证部署

```bash
# 测试服务
curl -I http://[公网IP]:[服务端口号]

# 或测试 Nginx
# curl -I http://[公网IP]

# 查看日志
journalctl -u [项目名称] -f
```

## 5. 常用命令速查

```bash
# SSH 连接
ssh root@[公网IP]

# 上传文件
scp file root@[公网IP]:/opt/

# 下载文件
scp root@[公网IP]:/opt/file ./

# 查看服务状态
systemctl status [项目名称]

# 重启服务
systemctl restart [项目名称]

# 查看日志
journalctl -u [项目名称] -f

# 查看端口占用
ss -tunlp | grep [服务端口号]

# 查看系统资源
htop
```
